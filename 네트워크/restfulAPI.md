Q RESTfulAPI
REST한 방식으로 클라이언트와 서버간 상호 데이터 교환을 하는 API이며, 서로간에 stateless한 특징을 가지는 API입니다.

 REST란
HTTP를 잘사용하기위해, URI와 HTTP메소드를 사용해서, URL로 어떤 자원에 접근할 것인지, 메소드로 어떤 행위를 할것인지 표현하여 설계된 API를 말합니다.

Q API를 고성능으로 만들기 위해서 하는 고민?

1. 캐싱
 API요청에 캐싱시간을 설정하여 특정시간만큼 데이터를 저장해 재요청을 방지하여 더 빠르게 동작가능하게 만듬

2. 상태유지
수십개의 API 요청 중 변경되는 소수의 데이터만 요청에 응답하고 고객의 재요청에 대응해 상태 유지를 시킨다.

3. 압축
데이터 전송 용량이 감소하여 속도가 올라가고 운영비용이 감소된다 

4. Hard 리퀘스트
get만 조회 요청으로 사용할 경우 캐싱에 따라 서버에서 304 not modified 리스폰스를 보내줄 수 있지만 이과정도 생략하고 오직 헤더만 받고자하면 Hard리퀘스트를 보내 사용자의 데이터가 변화가 있는지 체크 가능

REST API는 아키텍처의 제약 조건을 지키는 api이다. 최초에는 http의 버전 업 등을 기존 버전에 문제가 생기지 않도록 고민하다가 나온 방식이다. 따라서 REST API의 특징은 클라이언트와 서버가 서로 연결되어있지 않다는 것이다.
응용 방식은 보통 말하는 CRUD를 REST API만의 API 형식으로 만들어 쿼리 생성이 아닌 인터페이스에서 조작하는 것만으로 CRUD가 가능하다.

고성능 API에 대한 고민은 쿼리 조회가 복잡하지 않도록 소스 최적화, REST API의 결과를 캐싱 저장(스프링 3.1부터 제공) 등이 생각난다.
