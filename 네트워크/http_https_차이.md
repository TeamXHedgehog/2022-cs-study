```
Http는 서버, 클라이언트 간의 통신규약 프로토콜이다. 처음에는 html 전송목적으로 만들어졌으나 현재는 JASON 등 모든 파일을 전송한다.
http는 클라이언트 request가 있을 때 서버가 response 하는 단방향 통신이다.

https는 http에Secure를 붙인 용어로 http의 보안을 강화시키기 위해 개발됐다.
CA(인증기관)라는 3자 기관에서 서버가 안전한지 도메인과 공개키 등을 포함한 인증을 해 서버에 제공하여
클라이언트와 서버 간 인증키 등의 암복호화를 통해 데이터를 송수신하게 된다.


클라이언트는 서버에 통신 요청을 보내고 서버는 지원 가능한 공개키, 알고리즘, TLS 버전 등을 전달하고
CA로부터 제공받은 인증서를 클라이언트에 보냅니다.

클라이언트는 프리 마스터키를 공개키 방식으로 사용하여 전송하고 서버는 전송받은 프리 마스터키를 복호화한 후 
대칭키로 승격하여 데이터를 송수신하게 된다.
```

